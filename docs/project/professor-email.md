# Professor Email Draft

Subject: Project Proposal: Kinyarwanda Language Learning LM

Dear Professor [Name],

I hope you are doing well. After discussing ideas with classmates, I wanted to
share a proposed project direction for the rest of the course: building a
Kinyarwanda language learning LM/tutor that helps students learn Kinyarwanda
through explanations, examples, exercises, and sentence correction.

The idea is to connect the class material directly to a meaningful language
project. We would follow the spirit of Stanford CS336: Language Modeling from
Scratch by implementing the core pieces ourselves: BPE tokenization, a
decoder-only Transformer language model, cross-entropy loss, AdamW, the training
loop, checkpointing, sampling, and perplexity evaluation.

At the same time, we want the final demo to be useful, not only educational. So
the plan has two tracks: (1) a from-scratch small LM so we understand the full
pipeline, and (2) a practical Kinyarwanda learning assistant using
retrieval-augmented generation and/or light fine-tuning of a base model if time,
data, and compute allow.

The proposed final deliverables by August 14 are: a working code repository, a
trained Kinyarwanda tokenizer with analysis, a small from-scratch LM baseline
with loss/perplexity results, a Kinyarwanda tutor prototype, a data card/model
card, and a final demo/presentation.

We would be grateful for your feedback on five points:

1. Is this scope appropriate for the class timeline?
2. Which data sources are acceptable for training, reference, or evaluation?
3. Are there copyright or permission constraints we should be especially careful
   about?
4. What compute expectations should we assume?
5. What evaluation evidence would make this a strong final project?

Thank you, and I would love to hear your thoughts.

Best,

Jonathan
