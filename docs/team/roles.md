# Team Roles

Assign one primary owner and one backup for each area. Owners coordinate the
work; they do not have to do everything alone.

This is the starter split. Adjust it after the first team check-in if someone
has stronger availability or experience for a role.

## Responsibilities

### Project Lead / Coordinator

- run short check-ins,
- keep scope realistic,
- communicate with professor,
- maintain weekly demo plan,
- make sure blockers are visible.

### Data Lead

- find sources,
- record licenses and permissions,
- separate raw/interim/processed data,
- write cleaning notes,
- draft the data card.

### Tokenizer Lead

- own BPE tokenizer path,
- choose tokenizer evaluation examples,
- compare segmentation quality,
- report tokens-per-word and fragmentation.

### Modeling Lead

- own Transformer LM path,
- create tiny training configs,
- track loss/perplexity,
- handle checkpoints and sampling.

### Tutor / UX Lead

- define tutor capabilities,
- organize approved lesson notes,
- build retrieval-first flow,
- prepare demo tasks.

### Evaluation Lead

- create human evaluation rubric,
- collect pass/fail notes,
- track error categories,
- summarize final results.

### Documentation Lead

- keep project docs current,
- collect weekly updates,
- prepare report and presentation materials.

## Assignment Rule

No task should have only one person who understands it. Every critical path
needs a backup reviewer.
